# Conductor Linux

Десктопна апка під Linux для запуску **паралельних сесій Claude Code**, кожна — в ізольованому
**git worktree**. Linux-клон [conductor.build](https://www.conductor.build/).

Кожен воркспейс — це окремий checkout твого репозиторію (git worktree) з власною гілкою, власною
сесією `claude` і власними ресурсами (БД, порт тощо через скрипти). Працюй над кількома задачами
паралельно, не плутаючи їх між собою.

## Можливості

- 🗂️ **Воркспейси на git worktree** — ізольований checkout + власна гілка на кожну задачу.
- 🌿 **Вибір базової гілки при створенні** — динамічний список локальних і remote-гілок (із попереднім
  `git fetch`) зі швидким пошуком; нова гілка створюється саме від обраної.
- ✍️ **Повна назва гілки** — одне поле для назви воркспейсу/гілки (без примусового префікса) з
  валідацією на дублікати.
- 🤖 **Сесія Claude на воркспейс** — вбудований термінал (`node-pty` + `xterm.js`) з інтерактивним `claude`;
  сесії **автоматично відновлюються** при перезапуску апки.
- 💻 **Вкладка «Термінал»** — вільна інтерактивна shell-консоль у директорії воркспейсу для будь-яких команд.
- ⚙️ **Скрипти setup / run / archive** — налаштовувані шляхи; запускаються з env-змінними `CONDUCTOR_*`.
- 🚀 **Неблокуючий UI** — setup і archive виконуються у фоні (бейджі «⏳ налаштування…», «📦 архівується…»);
  можна одразу працювати в інших воркспейсах.
- ▶️⏹️ **Кнопка Run/Stop** — запуск і зупинка застосунку у воркспейсі однією кнопкою (вбиває всю групу
  процесів, тож dev-сервер не лишається висіти).
- 🌐 **Відкрити у браузері** — відкриває `http://localhost:<порт>` запущеного застосунку (активна під час Run).
- 🧩 **Відкрити в IDE** — відкриває worktree в обраному редакторі (команда задається в Налаштуваннях).
- 🗄️ **Архів воркспейсів** — архівовані можна **повернути** (відтворює worktree й заново проганяє setup)
  або **видалити назавжди** (разом із git-гілкою).
- 🧭 **Видимість гілок** — у шапці показано поточну гілку (наживо) і базову, від якої створено воркспейс.
- 📋 **Копіювання та контекстне меню** — правий клік у будь-якій консолі → «Копіювати»/«Вставити»;
  у read-only вкладці «Скрипти» виділення копіюється, а ввід не впливає на процес.
- 🧠 **Памʼять вкладок** — кожен воркспейс памʼятає, на якій вкладці ти був.
- 🔢 **Налаштовуваний початковий порт** — воркспейси отримують унікальні порти, починаючи з заданого.

## Інтерфейс

- **Зліва** — список воркспейсів, кнопка «➕ Новий воркспейс» (вгорі) і «🗄 Архів» (внизу).
- **По центру** — три вкладки:
  - **Claude** — інтерактивна сесія `claude` у директорії воркспейсу.
  - **Термінал** — вільна shell-консоль у директорії воркспейсу.
  - **Скрипти** — вивід setup/run/archive (read-only, але з копіюванням).
- **Згори** — назва/шлях воркспейсу, поточна/базова гілка, кнопки **Run/Stop**, **🌐 У браузері**,
  **🧩 Відкрити в IDE**, **Архівувати**.
- **⚙ Налаштування** — репозиторій, директорія для worktree, початковий порт, IDE-команда і три скрипти.

## Як це працює

| Дія | Що відбувається |
|-----|-----------------|
| Створення воркспейсу | вибираєш базову гілку + назву → `git worktree add <worktreesDir>/<slug> -b <назва> <база>` → у фоні: **setup**-скрипт → старт сесії `claude`. |
| **Run / Stop** | Run запускає **run**-скрипт у воркспейсі; кнопка стає **Stop** і зупиняє процес (разом із дочірніми). |
| **Архівувати** | зупиняє запущений застосунок → **archive**-скрипт → завершення PTY → `git worktree remove --force`; воркспейс лишається в **Архіві** (гілка зберігається). |
| **Повернути з архіву** | `git worktree add` на наявній гілці → заново проганяє **setup**-скрипт + стартує `claude`. |
| **Видалити з архіву** | прибирає worktree, **видаляє git-гілку** і запис воркспейсу (незворотно). |

Авторизація Claude береться з уже залогіненого `claude` CLI.

### Env-змінні для скриптів (сумісно з Conductor)

| Змінна | Значення |
|--------|----------|
| `CONDUCTOR_WORKSPACE_PATH` | абсолютний шлях директорії воркспейсу (worktree) |
| `CONDUCTOR_ROOT_PATH` | корінь головного репозиторію |
| `CONDUCTOR_WORKSPACE_NAME` | імʼя воркспейсу |
| `CONDUCTOR_PORT` | унікальний порт воркспейсу (від «початкового порту» з Налаштувань) |

У `scripts/` є приклади `setup.sh` / `run.sh` / `archive.sh` для перевірки. Скрипти живуть у твоєму
проєкті — апка лишається універсальною й нічого не хардкодить (напр. окрему БД на воркспейс створює
саме твій setup-скрипт через `CONDUCTOR_WORKSPACE_NAME`/`CONDUCTOR_PORT`).

## Архітектура

```
src/
  main/        # Electron main: вікно, git worktree, запуск скриптів, node-pty, персист стану
    index.ts ipc.ts store.ts env.ts git.ts ptyManager.ts workspaces.ts
  preload/     # contextBridge → typed window.api (contextIsolation)
  renderer/    # React + Vite: Sidebar, Toolbar, TerminalView, SettingsModal,
               #   NewWorkspaceModal, ArchivedModal, termRegistry, store
  shared/      # спільні типи
build/         # ресурси збірки (icon.png)
scripts/       # приклади setup/run/archive + install-desktop.sh
```

**Стек:** Electron + React + TypeScript, `node-pty` + `@xterm/xterm`, `zustand`, `electron-vite`.

## Розробка

```bash
npm install          # автоматично ребілдить node-pty під ABI Electron (postinstall)
npm run dev          # режим розробки
npm run build        # збірка main/preload/renderer у out/
npm run dist         # збірка AppImage/deb (electron-builder)
```

Потрібні: **Node 20+**, `git`, встановлений CLI **`claude`** у `PATH`.

## Встановлення (запуск як звичайної програми)

Після `npm run dist` зареєструй AppImage у меню програм (іконка + .desktop):

```bash
bash scripts/install-desktop.sh
```

Скрипт ставить іконку, створює `~/.local/share/applications/conductor-linux.desktop` (з `--no-sandbox`,
бо AppImage не може користуватись SUID-сендбоксом Chromium) і робить AppImage виконуваним. Далі шукай
**«Conductor»** у меню програм. Якщо переміщуєш AppImage — перезапусти скрипт із новим шляхом.

## Налаштування

При першому запуску відкриється вікно Налаштувань. Вкажи:

- **Репозиторій** — головний git-репозиторій, з якого створюються worktree.
- **Директорія для worktree** — де зберігати воркспейси (дефолт `~/.conductor-linux/worktrees`).
- **Початковий порт** — з якого числа роздавати `CONDUCTOR_PORT` (дефолт `3002`).
- **IDE** — команда для відкриття воркспейсу (напр. `code`, `cursor`, `subl`).
- **Setup / Run / Archive** — шляхи до скриптів (опційно).
